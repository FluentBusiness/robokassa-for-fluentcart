# Robokassa for FluentCart

Официальная интеграция платежного шлюза [Robokassa](https://robokassa.com/) для плагина электронной коммерции [FluentCart](https://fluentbusiness.pro/).

Плагин обеспечивает бесшовную интеграцию приема платежей, фискализацию по 54-ФЗ, поддержку подписок (рекуррентных платежей), холдирование средств и расширенные маркетинговые инструменты (оплата частями, рассрочка).

---

## 🛠 Технический анализ и архитектура

Плагин реализован с использованием объектно-ориентированного подхода и встроенных API FluentCart (`AbstractPaymentGateway`, `AbstractSubscriptionModule`).

### Структура файлов и папок

Модуль имеет следующую структуру директорий, где бизнес-логика разделена на функциональные блоки:

```text
robokassa-for-fluentcart/
├── assets/                  # Стили CSS и JS-скрипты для фронтенда и админки
├── data/                    # Директория для текстовых логов работы API
├── includes/                # Заголовочный каталог с классами PHP
│   ├── API/                 # Классы взаимодействия с API Robokassa
│   │   ├── AgentManager.php # Управление агентскими товарами
│   │   ├── RobokassaPayAPI.php # Формирование запросов к боевому API 
│   │   ├── TaxManager.php   # Управление налоговыми ставками (54-ФЗ)
│   │   └── ...              # Другие классы (управление СМС, методы, хелперы)
│   ├── Settings/            # Настройки плагина в админ-панели FluentCart
│   │   └── RobokassaSettingsBase.php 
│   ├── Webhook/             # Обработка асинхронных уведомлений (коллбэков)
│   │   └── RobokassaWebhookHandler.php
│   ├── RobokassaGateway.php       # Главный класс интеграции (шлюз оплаты)
│   ├── RobokassaSubscriptions.php # Обработчик рекуррентных платежей
│   └── RobokassaPaymentWidget.php # Виджет оплаты долями/в кредит для товаров
├── languages/               # Файлы локализации плагина (.pot-шаблоны)
└── robokassa-for-fluent-cart.php  # Точка входа в WP плагин (Bootstrap)
```

### Ключевые архитектурные компоненты

- **`RobokassaGateway`**: Основной класс шлюза. Регистрирует метод оплаты, обрабатывает флоу оплаты (редирект или Iframe), добавляет хуки для изменения статусов заказов и холдирования. Подгружает JS и CSS.
- **`API/RobokassaPayAPI`**: Ядро взаимодействия с API Robokassa. Генерирует подписи (MD5/SHA), формирует XML/JSON запросы для инициализации оплаты, создания токенов подписок, отправки SMS и получения доступных валют.
- **`RobokassaSubscriptions`**: Реализация логики рекуррентных платежей. Работает через крон-задачи WordPress (`robokassa_recurring_payment`). Обрабатывает renewal-заказы FluentCart и автоматически инициирует списания средств по сохраненным InvoiceID через end-point `auth.robokassa.ru/Merchant/Recurring`.
- **`Webhook/RobokassaWebhookHandler`**: Обработчик асинхронных уведомлений (IPN) от Robokassa о статусах платежей (Success/Fail). Валидирует подпись и изменяет статус транзакции и заказа в FluentCart.
- **`Settings/RobokassaSettingsBase`**: Интеграция с подсистемой настроек FluentCart в виде вкладок. Хранит конфигурации магазина, налогов, виджетов и SMS-уведомлений.

### Реализованный функционал под капотом

1. **Поддержка Iframe и Standard (Direct) оплаты:**
   - Плагин поддерживает перенаправление пользователя на платежную страницу (Standard).
   - Может рендерить форму во всплывающем (iframe) окне без ухода с сайта, передавая payload через JS-события.
   - Поддерживает прямую интеграцию дополнительных шлюзов (Яндекс.Сплит, Мокка, Подели, ОТП-кредит).
2. **Двухэтапная оплата (Холдирование):** Заморозка средств с последующим подтверждением или отменой. Завязан на хуки изменения статусов заказа FluentCart и Ajax-обработчики.
3. **Фискализация 54-ФЗ и ФФД 1.2:**
   - Формирование массива `receipt` с передачей налоговых ставок, признаков способа и предмета расчета (в том числе для отдельной сущности - "доставки").
   - Регистрация "Второго чека" в момент передачи товара. Плагин использует хук `fluent_cart/order_status_changed` для автоматической отправки 2-го чека при переводе заказа в заданный статус окончания сделки.
4. **Агентская схема:** Поддержка передачи данных об агентах на уровне отдельных позиций (через `AgentManager`). Позволяет указывать ИНН поставщика, тип агента, телефон.
5. **REST API-эндпоинты:** Регистрация собственных эндпоинтов WP REST API (маршрут `/robokassa/v1/status`) для поллинга (polling) статусов транзакций на фронтенде после iframe-оплаты.
6. **SMS-Уведомления:** Использование API `services.robokassa.ru/SMS` для отправки сервисных сообщений покупателям. Текст поддерживает переменные `{address}`, `{fio}`, `{order_number}`.

## 📖 Руководство пользователя

Данное руководство предназначено для владельцев интернет-магазинов и администраторов сайтов.

### 1. Поддерживаемые возможности

* **Прием оплаты:** банковские карты, СБП (QR-коды), электронные кошельки.
- **Холдирование:** заморозка средств на карте до подтверждения заказа (отличное решение при ручной проверке наличия товара перед списанием).
- **Рекуррентные платежи:** организация регулярных списаний по подписке (через модуль FluentCart Subscriptions).
- **Маркетинговые инструменты (BNPL/Кредит):** покупка частями через Яндекс Сплит, Подели, рассрочка от ОТП Банка и сервис Мокка.
- **Фискализация (54-ФЗ):** автоматическая передача данных в облачную кассу или сервис «Робочеки». *Внимание: для цифровых товаров адрес доставки отключается.*
- **SMS-оповещения:** информирование клиента об успешной оплате и статусе заказа.

### 2. Первоначальная активация и интеграция

Для начала работы необходимо активировать модуль в административной панели: **FluentCart Pro -> Настройки -> Настройки оплаты -> Robokassa**.

**Шаг 1: Основные настройки**

1. Переведите переключатель **Активация оплаты** в положение «Включено».
2. Выберите **Страну магазина** (Россия или Казахстан).
3. Введите **Идентификатор магазина** (MerchantLogin), полученный при регистрации в Robokassa.

**Шаг 2: Настройка технических адресов**
Для того чтобы Robokassa могла «сообщить» сайту об успешной или отмененной оплате, скопируйте уникальные URL-адреса из блока «Помощь» в настройках плагина и вставьте их в Личном Кабинете Robokassa (раздел "Технические настройки"):
- **ResultURL**
- **SuccessURL**
- **FailURL**
*(Установите метод отсылки данных — **POST**, алгоритм расчета хеша — **MD5**).*

**Шаг 3: Авторизационные данные (Пароли)**
Введите Пароль магазина №1 (используется для подписи платежа) и Пароль магазина №2 (для проверки результата) из ЛК Робокассы.

### 3. Режимы работы: Тестовый и Боевой

Перед запуском реальных продаж обязательно проведите проверку в тестовом режиме.

1. Во вкладке **Тестовое** включите **Тестовый режим**.
2. Введите Тестовые пароли №1 и №2 из ЛК Робокассы.
В этом режиме реальные деньги не списываются, транзакции отображаются в спецразделе тестовых операций ЛК.

### 4. Выбор интерфейса: Standard vs Iframe

Во вкладке основных настроек вы можете выбрать "Включить iframe":

| Характеристика | Standard (Стандарт) | Iframe (Окно) |
| --- | --- | --- |
| Процесс | Перенаправление (редирект) на сайт Robokassa | Вызов формы поверх вашего сайта |
| Методы оплаты | Все (СБП, карты, рассрочки) | В основном карты / Apple, Samsung Pay / Qiwi |
| Адаптивность | Подстраивается под устройство | Требует минимум поддержки JS фронтендом |

*Внимание: Iframe-режим может конфликтовать со сторонними скриптами темы. Если страница "Успех" не прогружается корректно, используйте Standard.*

### 5. Маркетинговые виджеты и рассрочки

Для повышения конверсии (особенно при высоких средних чеках) включите Yandex Split, Мокка или ОТП-кредит.
Во вкладке **Виджет и бейдж** можно активировать показ бейджа в карточке товара, который подскажет пользователю возможность купить товар в рассрочку. Данный виджет гибко настраивается по цветам, размеру и теме.

### 6. Фискализация (54-ФЗ)

1. Выберите систему налогообложения (СНО).
2. Выберите источник налоговой ставки (общая или индивидуальная из карточки товара).
3. **Двухэтапная фискализация (Второй чек):** Если вы принимаете предоплату, вы обязаны выбить второй чек (полный расчет) при физическом отпуске товара. Выберите статус заказа, при переходе в который плагин автоматически выбьет финальный второй чек (например "Выполнен").

### 7. Холдирование средств

Холдирование позволяет удерживать деньги на карте без фактического зачисления вам на счет, пока вы не убедитесь, что товар доступен для отправки.
- В настройках плагина выберите "Включено". Заморозка действует до 7 дней.
- Во FluentCart заказ получает статус `на удержание` (`on-hold`).
- Чтобы забрать/списать деньги на свой счет, нажмите "Холдирование" внутри заказа. Чтобы вернуть деньги клиенту, измените статус заказа на "Отменен".

### 8. Рекуррентные платежи (Подписки)

Создавайте во FluentCart товары в формате "Подписка". Обновляемые платежи будут сниматься автоматически.
В настройках шлюза Робокассы обязательно заполните текст оферты – она будет показана при привязке карты.
*Обратите внимание: кнопки "Синхронизировать" или "Приостановить" работают только для отмены подписки (через удаление крон-задач wp_schedule)*.

### 9. Агентские товары

Включите опцию "Продаете ли вы агентский товар?", если продаете товары от лица партнеров-поставщиков. В карточке товара появятся обязательные поля:
- Тип агента (Например: Банковский платежный агент)
- ИНН поставщика
- Наименование поставщика
- Телефон

Этих данных достаточно для выбивания фискальных чеков со стороны Robokassa.

### 10. Популярные ошибки (FAQ)

* **Ошибка 25:** Магазин не активирован в Robokassa. Анкета на модерации.
- **Ошибка 26:** Магазин не найден (опечатка или пробелы в MerchantLogin).
- **Ошибка 29:** Неверная подпись. Перепутаны пароль №1 и №2, либо пытаетесь использовать боевые пароли при галочке "Тестовый режим".
- **Ошибка 40:** Повторная оплата счета с выданным ID. Заказ уже был оплачен или ID дублируется.
- При обращениях в службу заботы обязательно называйте свой MerchantLogin и номер заказа в системе.`
