# Robokassa for FluentCart

Официальная интеграция платежного шлюза [Robokassa](https://robokassa.com/) для плагина электронной коммерции [FluentCart](https://fluentbusiness.pro/).

Плагин обеспечивает бесшовную интеграцию приема платежей, фискализацию по 54-ФЗ, поддержку подписок (рекуррентных платежей), холдирование средств и расширенные маркетинговые инструменты (оплата частями, рассрочка).

---

## 📋 Системные требования

- **WordPress:** 5.8 или выше
- **FluentCart:** 1.2.5 или выше (плагин проверяет версию при активации)
- **PHP:** 7.4 или выше
- **Аккаунт Robokassa:** зарегистрированный и активированный магазин (Россия или Казахстан)

---

## ⚙️ Установка

1. Скопируйте папку плагина в директорию `/wp-content/plugins/`.
2. Активируйте плагин в разделе **Плагины** административной панели WordPress.
3. При отсутствии FluentCart 1.2.5+ плагин не активируется и выведет уведомление об ошибке.
4. После активации перейдите в **FluentCart Pro → Настройки → Настройки оплаты → Robokassa** для конфигурации.

## 🛠 Технический анализ и архитектура

Плагин реализован с использованием объектно-ориентированного подхода и встроенных API FluentCart (`AbstractPaymentGateway`, `AbstractSubscriptionModule`).

### Структура файлов и папок

Модуль имеет следующую структуру директорий, где бизнес-логика разделена на функциональные блоки:

```text
robokassa-for-fluentcart/
├── assets/                  # Стили CSS и JS-скрипты для фронтенда и админки
├── data/                    # Директория для текстовых логов работы API
├── includes/                # Заголовочный каталог с классами PHP
│   ├── API/                 # Классы взаимодействия с API Robokassa
│   │   ├── AgentManager.php # Управление агентскими товарами
│   │   ├── RobokassaPayAPI.php # Формирование запросов к боевому API
│   │   ├── RobokassaHelper.php # Константы и справочные данные (культуры, методы, объекты расчёта)
│   │   ├── RobokassaMethods.php # Дополнительные методы взаимодействия с API
│   │   ├── RobokassaSms.php # Отправка SMS через services.robokassa.ru
│   │   ├── RoboDataBase.php # Работа с базой данных (логирование транзакций)
│   │   ├── TaxManager.php   # Управление налоговыми ставками (54-ФЗ)
│   │   └── Util.php         # Вспомогательные утилиты (URL-хелперы)
│   ├── Settings/            # Настройки плагина в админ-панели FluentCart
│   │   └── RobokassaSettingsBase.php
│   ├── Webhook/             # Обработка асинхронных уведомлений (коллбэков)
│   │   └── RobokassaWebhookHandler.php
│   ├── RobokassaGateway.php       # Главный класс интеграции (шлюз оплаты)
│   ├── RobokassaSubscriptions.php # Обработчик рекуррентных платежей
│   └── RobokassaPaymentWidget.php # Виджет оплаты долями/в кредит для товаров
├── languages/               # Файлы локализации плагина (.pot-шаблоны)
└── robokassa-for-fluent-cart.php  # Точка входа в WP плагин (Bootstrap)
```

### Ключевые архитектурные компоненты

- **`RobokassaGateway`**: Основной класс шлюза. Регистрирует метод оплаты через хук `fluent_cart/register_payment_methods`, объявляет поддерживаемые возможности (`payment`, `refund`, `webhook`, `subscriptions`), обрабатывает флоу оплаты (редирект или Iframe), добавляет хуки для изменения статусов заказов и холдирования. Подгружает JS и CSS как для фронтенда, так и для админ-панели.
- **`API/RobokassaPayAPI`**: Ядро взаимодействия с API Robokassa. Генерирует подписи с поддержкой алгоритмов MD5, RIPEMD160, SHA1, SHA256, SHA384 и SHA512. Формирует XML/JSON-запросы для инициализации оплаты (включая параметры холдирования `StepByStep`), создания токенов подписок, отправки SMS и получения доступных валют через `GetCurrencies`. Поддерживает endpoint как `auth.robokassa.ru`, так и `auth.robokassa.kz` (Казахстан).
- **`RobokassaSubscriptions`**: Реализация логики рекуррентных платежей. Работает через крон-задачи WordPress (`robokassa_recurring_payment`). Обрабатывает renewal-заказы FluentCart через `SubscriptionService::recordRenewalPayment()`, формирует фискальный чек и автоматически инициирует списания средств по сохраненным `InvoiceID` через endpoint `auth.robokassa.ru/Merchant/Recurring`.
- **`Webhook/RobokassaWebhookHandler`**: Обработчик асинхронных уведомлений (IPN) от Robokassa об успешном статусе (Success) платежа. Валидирует подпись и изменяет статус транзакции и заказа в FluentCart.
- **`Settings/RobokassaSettingsBase`**: Интеграция с подсистемой настроек FluentCart в виде вкладок. Хранит конфигурации магазина, налогов, виджетов и SMS-уведомлений. Содержит встроенную форму регистрации нового магазина Robokassa через iframe (`reg2.robokassa.ru`), которая автоматически передаёт URL-адреса обратных вызовов сайта через `postMessage`.
- **`API/TaxManager`**: Управление налоговыми ставками на уровне позиций заказа. Поддерживает получение ставки из глобальных настроек плагина или из метаданных конкретного товара (`_robokassa_tax_rate`). Добавляет виджет выбора налога в карточку товара через хук `fluent_cart/product_updated`.

### Реализованный функционал под капотом

1. **Поддержка Iframe и Standard (Direct) оплаты:**

   - Плагин поддерживает перенаправление пользователя на платежную страницу (Standard) — автоматически отправляет POST-форму с задержкой.
   - Может рендерить форму во всплывающем (iframe) окне без ухода с сайта, передавая payload через JS-события (`CustomEvent("robokassa:params")`).
   - Поддерживает прямую интеграцию дополнительных шлюзов: Яндекс Сплит (`YandexPaySplit`), Мокка (`Mokka`), Подели (`Podeli`), ОТП-кредит (`OTP`) через SDK `DirectPayment.js`.

2. **Двухэтапная оплата (Холдирование):** Заморозка средств с последующим подтверждением или отменой. Передаёт параметр `StepByStep=true` при инициализации платежа. Завязан на хуки изменения статусов заказа FluentCart и Ajax-обработчики.

3. **Фискализация 54-ФЗ и ФФД 1.2:**

   - Формирование массива `receipt` с передачей налоговых ставок, признаков способа и предмета расчёта (в том числе для отдельной сущности — «доставки»).
   - Поддерживаемые ставки НДС: `none`, `vat0`, `vat5`, `vat7`, `vat8`, `vat10`, `vat12`, `vat20`, `vat105`, `vat107`, `vat110`, `vat118`. Ставки `vat8` и `vat12` предназначены для Казахстана.
   - Регистрация «Второго чека» в момент передачи товара. Плагин использует хук `fluent_cart/order_status_changed` для автоматической отправки второго чека при переводе заказа в заданный статус.
   - Налоговая ставка может быть задана глобально в настройках плагина или переопределена индивидуально для каждого товара через метаданные.

4. **Агентская схема:** Поддержка передачи данных об агентах на уровне отдельных позиций (через `AgentManager`). Позволяет указывать ИНН поставщика, тип агента, наименование, телефон. **По агентской схеме доступно не для всех схем фискализации — на Робочеках не будет доступно, но будет на [Robokassa Online](https://robokassa.com/online-check/robokassa-online/)**

5. **REST API-эндпоинты:** Регистрация собственных эндпоинтов WP REST API (маршрут `/robokassa/v1/status`) для поллинга (polling) статусов транзакций на фронтенде после iframe-оплаты. Возвращает статус, сумму, валюту, ссылку на чек и URL для редиректа (успех/отказ).

6. **SMS-уведомления:** Использование API `services.robokassa.ru/SMS` для отправки двух типов сервисных сообщений: об успешной оплате и о завершении заказа. Текст поддерживает переменные `{address}`, `{fio}`, `{order_number}`. Доступна транслитерация текста.

7. **Встроенная форма регистрации:** Вкладка «Регистрация» в настройках плагина содержит iframe с формой `reg2.robokassa.ru`, которая автоматически передаёт URL-адреса колбэков сайта (`result_url`, `success_url`, `fail_url`) через `postMessage` — регистрация нового магазина прямо из WP-админки.

## 📖 Руководство пользователя

Данное руководство предназначено для владельцев интернет-магазинов и администраторов сайтов.

### 1. Поддерживаемые возможности

- **Прием оплаты:** банковские карты, СБП (QR-коды), электронные кошельки.
- **Холдирование:** заморозка средств на карте до подтверждения заказа (отличное решение при ручной проверке наличия товара перед списанием).
- **Рекуррентные платежи:** организация регулярных списаний по подписке (через модуль FluentCart Subscriptions).
- **Маркетинговые инструменты (BNPL/Кредит):** покупка частями через Яндекс Сплит, Подели, рассрочка от ОТП Банка и сервис Мокка.
- **Фискализация (54-ФЗ):** автоматическая передача данных в облачную кассу или сервис «Робочеки». *Внимание: для цифровых товаров адрес доставки отключается.*
- **SMS-оповещения:** информирование клиента об успешной оплате и статусе заказа.
- **Поддержка Казахстана:** отдельный endpoint `auth.robokassa.kz` и ставки НДС 8% и 12%.

### 2. Первоначальная активация и интеграция

Для начала работы необходимо активировать модуль в административной панели: **FluentCart Pro → Настройки → Настройки оплаты → Robokassa**.

**Шаг 1: Основные настройки**

1. Переведите переключатель **Активация оплаты** в положение «Включено».
2. Выберите **Страну магазина** (Россия или Казахстан).
3. Введите **Идентификатор магазина** (MerchantLogin), полученный при регистрации в Robokassa.

**Шаг 2: Настройка технических адресов**
Для того чтобы Robokassa могла «сообщить» сайту об успешной или отменённой оплате, скопируйте уникальные URL-адреса из блока «Помощь» в настройках плагина и вставьте их в Личном Кабинете Robokassa (раздел «Технические настройки»):

- **ResultURL** — `https://ваш-сайт.ru/?robokassa=result`
- **SuccessURL** — `https://ваш-сайт.ru/?robokassa=success`
- **FailURL** — `https://ваш-сайт.ru/?robokassa=fail`

*(Установите метод отсылки данных — **POST**, алгоритм расчёта хеша — **MD5**).*

**Шаг 3: Авторизационные данные (Пароли)**
Введите Пароль магазина №1 (используется для подписи платежа) и Пароль магазина №2 (для проверки результата) из ЛК Робокассы.

**Шаг 4 (опционально): Язык интерфейса**
В настройках можно выбрать язык платёжной страницы Robokassa: русский, английский или автоопределение.

### 3. Режимы работы: Тестовый и Боевой

Перед запуском реальных продаж обязательно проведите проверку в тестовом режиме.

1. Во вкладке **Тестовое** включите **Тестовый режим**.
2. Введите Тестовые пароли №1 и №2 из ЛК Робокассы.

В этом режиме реальные деньги не списываются, транзакции отображаются в спецразделе тестовых операций ЛК.

### 4. Выбор интерфейса: Standard vs Iframe

Во вкладке основных настроек вы можете выбрать «Включить iframe»:

| Характеристика | Standard (Стандарт) | Iframe (Окно) |
| --- | --- | --- |
| Процесс | Перенаправление (редирект) на сайт Robokassa | Вызов формы поверх вашего сайта |
| Методы оплаты | Все (СБП, карты, рассрочки) | В основном карты / Apple, Samsung Pay / Qiwi |
| Адаптивность | Подстраивается под устройство | Требует минимум поддержки JS фронтендом |
| Статус после оплаты | Стандартный редирект | Polling через REST API `/robokassa/v1/status` |

*Внимание: Iframe-режим может конфликтовать со сторонними скриптами темы. Если страница «Успех» не прогружается корректно, используйте Standard.*

### 5. Маркетинговые виджеты и рассрочки

Для повышения конверсии (особенно при высоких средних чеках) включите Yandex Split, Мокка или ОТП-кредит.
Во вкладке **Виджет и бейдж** можно активировать показ бейджа в карточке товара, который подскажет пользователю возможность купить товар в рассрочку.

**Параметры настройки виджета:**

| Параметр | Описание |
| --- | --- |
| **Компонент** | `widget` или `badge` |
| **Цветовая тема** | Светлая (`light`) или тёмная (`dark`) |
| **Размер** | `s` (малый) или `m` (средний) |
| **Тип оффера** | `bnpl`, `credit` или оба одновременно |
| **Показ логотипа** | Вкл/Выкл |
| **Скругление углов** | CSS-значение `borderRadius`, например `12px` |
| **Вторая строка описания** | Для виджета размера `m` |
| **Позиция описания** | `left` (слева) или `right` (справа) |

### 6. Фискализация (54-ФЗ)

1. Выберите систему налогообложения (СНО): ОСН, УСН доход, УСН доход минус расход, ЕНВД, ЕСН, Патент.
2. Выберите источник налоговой ставки:
   - **Из настроек плагина** — единая ставка для всех товаров.
   - **Из карточки товара** — индивидуальная ставка для каждого товара (если не задана, используется глобальная).
3. **Двухэтапная фискализация (Второй чек):** Если вы принимаете предоплату, вы обязаны выбить второй чек (полный расчёт) при физическом отпуске товара. Выберите статус заказа, при переходе в который плагин автоматически выбьет финальный второй чек (например «Выполнен»).
4. Для товаров и доставки можно раздельно указывать признаки предмета расчёта (`payment_object`) и способа расчёта (`payment_method`).

### 7. Холдирование средств

Холдирование позволяет удерживать деньги на карте без фактического зачисления вам на счёт, пока вы не убедитесь, что товар доступен для отправки.

- В настройках плагина выберите «Включено». Заморозка действует до 7 дней.
- Во FluentCart заказ получает статус `на удержание` (`on-hold`).
- Чтобы забрать/списать деньги на свой счёт, нажмите «Холдирование» внутри заказа. Чтобы вернуть деньги клиенту, измените статус заказа на «Отменён».

> *Услуга доступна только по предварительному согласованию с Robokassa. Работает исключительно при оплате банковской картой.*

### 8. Рекуррентные платежи (Подписки)

Создавайте во FluentCart товары в формате «Подписка». Обновляемые платежи будут сниматься автоматически через крон WordPress.
В настройках шлюза Робокассы обязательно заполните текст оферты — она будет показана при привязке карты.

*Обратите внимание: кнопки «Синхронизировать» или «Приостановить» работают только для отмены подписки (через удаление крон-задач `wp_schedule`).*

### 9. SMS-уведомления

Во вкладке **Уведомления** доступна настройка двух типов SMS-сообщений через `services.robokassa.ru`:

| Тип | Когда отправляется |
| --- | --- |
| **Уведомление об успешной оплате** | Сразу после подтверждения платежа |
| **Уведомление о завершении заказа** | При переводе заказа в финальный статус |

**Доступные переменные в тексте:**

- `{address}` — адрес доставки
- `{fio}` — ФИО покупателя
- `{order_number}` — номер заказа

Дополнительно можно включить **транслитерацию** текста для отправки на телефоны без поддержки кириллицы.

### 10. Агентские товары

Включите опцию «Продаёте ли вы агентский товар?», если продаёте товары от лица партнёров-поставщиков. В карточке товара появятся обязательные поля:

- Тип агента (Например: Банковский платёжный агент)
- ИНН поставщика
- Наименование поставщика
- Телефон

Этих данных достаточно для выбивания фискальных чеков со стороны Robokassa.

### 11. Регистрация нового магазина

Вкладка **Регистрация** в настройках плагина позволяет зарегистрировать новый магазин в Robokassa прямо из административной панели WordPress — без необходимости открывать сайт Robokassa отдельно. Форма автоматически заполняет технические URL-адреса вашего сайта.

### 12. Популярные ошибки (FAQ)

- **Ошибка 25:** Магазин не активирован в Robokassa. Анкета на модерации.
- **Ошибка 26:** Магазин не найден (опечатка или пробелы в MerchantLogin).
- **Ошибка 29:** Неверная подпись. Перепутаны пароль №1 и №2, либо пытаетесь использовать боевые пароли при галочке «Тестовый режим».
- **Ошибка 40:** Повторная оплата счёта с выданным ID. Заказ уже был оплачен или ID дублируется.
- При обращениях в службу заботы обязательно называйте свой MerchantLogin и номер заказа в системе.

[Типовые ошибки интерфейса оплаты Robokassa](https://docs.robokassa.ru/ru/pay-interface#payment-interface-errors)
