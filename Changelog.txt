# Changelog

Все заметные изменения в проекте Robokassa for FluentCart будут документироваться в этом файле.

## [0.1.0] - Первый релиз

### Добавлено
- **Базовая интеграция шлюза Robokassa**: 
  - Регистрация платежного метода шлюза оплаты в экосистеме FluentCart.
  - Поддержка двух режимов оплаты: Standard (с перенаправлением) и Iframe (вызов окна оплаты без ухода со страницы).
  - Тестовый и боевой режим с отдельными наборами паролей.
- **Поддержка рекуррентных платежей (Подписки)**: 
  - Полная поддержка продления платежей через планировщик WP-Cron (`robokassa_recurring_payment`).
  - Сохранение `InvoiceID` при первой оплате и автоматическое продление со стороны плагина без необходимости повторного ввода данных карты.
- **Фискализация 54-ФЗ (ФФД 1.2)**: 
  - Передача состава чека в "Робочеки".
  - Гибкая настройка системы налогообложения (СНО) и налоговых ставок на уровне всего магазина или отдельных товаров.
  - Функционал «Второго чека» (чека полного расчета) при изменении статуса заказа на завершенный (для тех, кто принимает предоплату).
- **Холдирование средств**: 
  - Заморозка денег на карте покупателя с ручным подтверждением или отменой списания из админки заказа FluentCart.
- **Интеграция сторонних маркетинговых инструментов**: 
  - Поддержка оплаты частями (BNPL): Яндекс Сплит, Подели.
  - Поддержка оплаты в рассрочку/кредит: Мокка, Кредит от ОТП Банка.
  - Информационный виджет и бейдж (с расчетом долей) для вывода на страницах товаров с гибкой настройкой дизайна.
- **Агентская схема**: 
  - Передача полей поставщика (ИНН, название, тип агента, телефон) для отдельных товаров (поддержка продаж агентских услуг).
- **SMS-уведомления**: 
  - Отправка SMS-оповещений покупателю при успешной оплате или завершении заказа через инфраструктуру Robokassa с поддержкой переменных шаблонов (`{fio}`, `{order_number}`, `{address}`).
- **Взаимодействие по API и Webhook**: 
  - Обработка callback-запросов от Робокассы для мгновенной смены статуса заказа.
  - Регистрация маршрута WP REST API `/robokassa/v1/status` для отслеживания (polling) статуса транзакции при Iframe-оплате.
- **Админ-настройки**: 
  - Удобная панель настроек шлюза во FluentCart, разбитая на удобные логические вкладки.
  - Возможность прямой регистрации магазина в Robokassa через Iframe в панели настроек.

### Безопасность и стабильность
- Адаптивная генерация MD5/SHA подписей для всех запросов к API.
- Проверка зависимостей: плагин автоматически проверяет наличие активированного ядра FluentCart версии 1.2.5+ и предупреждает администратора.
- Логирование: поддержка `debug_mode` для записи отладочной информации и `data/robokassa_DEBUG.txt`.
